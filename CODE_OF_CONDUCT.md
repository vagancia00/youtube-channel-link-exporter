# Code of Conduct

Project spaces should remain technical, respectful and useful.

## Expected behavior

- Discuss behavior, evidence and reproducible results rather than attacking people.
- Respect privacy and avoid publishing sensitive information.
- Keep examples synthetic when reporting parser issues.
- Disclose conflicts and security concerns responsibly.

## Unacceptable behavior

- Harassment, threats or discriminatory language.
- Publishing private information without permission.
- Submitting malicious code or deceptive instructions.
- Knowingly posting credentials, cookies, browser profiles or unsafe real-world fixtures.

Maintainers may edit, remove or reject contributions that violate this standard and may restrict participation when necessary to protect the project and its users.
